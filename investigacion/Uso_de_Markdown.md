# Investigación sobre el uso de Markdown

Markdown es un lenguaje de marcado ligero que permite dar formato al
texto de forma sencilla. A continuación se incluyen ejemplos de
encabezados, listas, tablas, enlaces e imágenes, así como las ventajas
de usar Markdown con GitHub.

------------------------------------------------------------------------

## 1. Encabezados

``` markdown
# Título nivel 1
## Título nivel 2
### Título nivel 3
#### Título nivel 4
##### Título nivel 5
###### Título nivel 6
```

------------------------------------------------------------------------

## 2. Listas

### Lista desordenada

``` markdown
- Manzanas
- Peras
- Uvas
```

### Lista ordenada

``` markdown
1. Paso uno
2. Paso dos
3. Paso tres
```

### Lista anidada

``` markdown
- Animales
  - Perro
  - Gato
    - Siames
    - Persa
```

------------------------------------------------------------------------

## 3. Tablas

``` markdown
| Nombre    | Edad | Ciudad     |
|-----------|------|------------|
| Ana       | 23   | Quito      |
| Marcos    | 30   | Guayaquil  |
| Valentina | 27   | Cuenca     |
```

------------------------------------------------------------------------

## 4. Enlaces

``` markdown
[Markdown Guide](https://www.markdownguide.org/)
```

------------------------------------------------------------------------

## 5. Imágenes

``` markdown
![Descripción](https://ejemplo.com/imagen.png)
```

------------------------------------------------------------------------

## 6. Ventajas de usar Markdown con GitHub

-   Perfecto para documentación.
-   Facilita el versionamiento.
-   Compatible con GitHub Pages.
-   Ligero, rápido y fácil de escribir.
-   Funciona en issues, pull requests y wikis.
-   Tiene un ecosistema ampliado (listas de tareas, referencias,
    resaltado de código).

### Ejemplo de lista de tareas:

``` markdown
- [x] Crear README
- [ ] Agregar documentación
- [ ] Subir cambios finales
```
